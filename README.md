# 🗺️ CityNavigatorAPP — Навигационная система для городских маршрутов

Курсовая работа по дисциплине **«Алгоритмизация и программирование»** (Направление: Искусственный интеллект, 1 курс, ТвГТУ).

Приложение для построения и анализа оптимальных маршрутов в городской транспортной сети на основе графовых алгоритмов.

---

## ✨ Функциональность

- **Загрузка графа**: Парсинг пользовательского текстового файла с описанием вершин (перекрёстков) и рёбер (дорог).
- **Алгоритм Дейкстры**: Поиск маршрута с минимальным весом (по расстоянию или по времени с учётом скорости).
- **Поиск в ширину (BFS)**: Поиск пути с минимальным количеством перекрёстков.
- **Альтернативные маршруты**: Поиск пути в обход самого длинного/медленного участка основного маршрута.
- **Визуализация**: Отрисовка графа, подсветка маршрута и автоматическое масштабирование карты.
- **Тестирование**: Покрытие кода модульными тестами >95% (NUnit 3 + Coverlet).

## 📖 Как пользоваться

1. **Запуск**: При старте программы выберите текстовый файл с описанием графа.
2. **Выбор точек**: В выпадающих списках `From` / `To` укажите начальную и конечную точку.
3. **Настройки**:
   - ☑ **«Самый быстрый путь»** — оптимизация по времени, а не по километражу.
   - ☑ **«Минимум перекрёстков»** — переключение на алгоритм BFS.
4. **Построение**: Нажмите **«Построить маршрут»**.
5. **Альтернатива**: После построения основного пути нажмите **«Другой путь»** для поиска объезда.

### 📄 Формат входного файла (`.txt`)

```text
# Вершина: V <ID> <X> <Y> <Название>
V 1 100 200 Москва
V 2 300 200 Тверь

# Ребро: E <From> <To> <Длина_км> [Скорость_км/ч]
E 1 2 170 90
```

| Параметр | Описание |
|----------|----------|
| `ID` | Уникальный идентификатор вершины (целое число) |
| `X`, `Y` | Координаты для визуализации |
| `Название` | Имя города/перекрёстка |
| `From`, `To` | ID соединяемых вершин |
| `Длина_км` | Протяжённость участка дороги |
| `Скорость_км/ч` | Средняя скорость (опционально, по умолчанию 45) |
