#  Навигационная система для городских маршрутов

Курсовая работа по дисциплине **«Алгоритмизация и программирование»** (Направление: Искусственный интеллект, 1 курс).

Проект представляет собой десктопное приложение для построения оптимальных маршрутов в городской среде на основе графовых алгоритмов. Реализованы методы поиска кратчайшего пути, построения альтернативных маршрутов и визуализация транспортной сети.

---

## Функциональность

- **Загрузка графа:** Парсинг пользовательского текстового файла с описанием вершин (перекрёстков) и рёбер (дорог).
- **Алгоритм Дейкстры:** Поиск маршрута с минимальным весом (по расстоянию или по времени с учётом скорости).
- **Поиск в ширину (BFS):** Поиск пути с минимальным количеством перекрёстков (минимум поворотов).
- **Альтернативные маршруты:** Поиск пути в обход самого длинного/медленного участка основного маршрута.
- **Визуализация:** Отрисовка графа, подсветка маршрута и масштабирование карты.
- **Тестирование:** Покрытие кода модульными тестами >95% (NUnit).

## Как пользоваться

1. **Запуск:** При открытии программы выберите текстовый файл с описанием графа (формат описан ниже).
2. **Выбор точек:** В выпадающих списках выберите начальную (`From`) и конечную (`To`) точки.
3. **Настройки:**
   - Галочка **«Самый быстрый путь»**: оптимизирует маршрут по времени (км/ч), а не по километражу.
   - Галочка **«Минимум перекрёстков»**: переключает алгоритм на BFS.
4. **Построение:** Нажмите кнопку **«Построить маршрут»**.
5. **Альтернатива:** После построения основного пути нажмите **«Другой путь»**, чтобы найти объездную дорогу.

### Формат входного файла (`.txt`)

Программа принимает файл в следующем формате:
- Строки с `V` описывают вершины: `V ID X Y Название`
- Строки с `E` описывают дороги: `E От Куда Длина Скорость`

**Пример файла:**
```text
V 1 100 200 Москва
V 2 300 200 Тверь
E 1 2 170 90
```

##                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     Архитектура

Проект разделён на два основных модуля для обеспечения чистоты кода и тестируемости:

1. **`NavigatorLogic`**: Библиотека классов (`.dll`). Содержит реализацию алгоритмов (`MainLogic`), структуры данных (`Vertex`, `Edge`, `Graph`) и парсер файлов.

3. **`NavigatorTests`**: Проект с модульными тестами NUnit.
